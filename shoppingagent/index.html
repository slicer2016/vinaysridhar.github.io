API-first Search vs. Agentic Commerce

I built this to explore the practical difference between traditional keyword-based search and the emerging Agent Commerce Protocol (ACP) — a new spec that lets AI agents interact directly with merchant systems. This prototype illustrates the differences between the prior paradigm and what the ACP can unlock. The same query runs through both approaches, retrieving ~40 products fetched from free catalog API. This is an example of waht an "ACP-enabled" shopping experience could look like - imagine having shoppers with different personas. Technically, you don't need such protocols to build these experiences, but it will help set up the shopping ecosystem to innovate faster with AI. Note: the checkout functionality is illustrative.

API Search
How it works

Products are fetched live from Google Shopping (via SerpAPI) for each query. Results are ranked by a simple popularity score: rating × log(1 + review_count). This rewards items that are both highly rated and widely reviewed — a product with 4.8★ from 3 reviews ranks below one with 4.6★ from 800 reviews.

Stack: Google Shopping → rating × log(1 + reviews) → ranked list

ACP World
Agentic Commerce
How it works

The same product catalog is fetched live for each query and passed in full to a Claude model (50 products, with ratings, reviews, and delivery info). Claude reasons over it and selects the top 5 matches. Clicking Buy Now on any pick opens an ACP-compliant checkout session — no redirect required.

Stack: Claude (claude-sonnet-4-6) → catalog reasoning → ACP-like checkout → Stripe (test)
